# PRD - 線上算命系統

## 1. 專案概述

### 背景與動機
現代人生活節奏快速，經常面臨工作、感情或生活上的壓力與抉擇。傳統的廟宇求籤或算命需要親自前往，對於忙碌的現代人來說門檻較高。因此，我們希望建立一個「線上算命系統」，讓使用者能隨時隨地透過網路進行抽籤、占卜，不僅能提供心靈上的慰藉與指引，還能將傳統信仰文化與數位生活結合。

### 目標用戶
- 尋求心靈撫慰、近期面臨人生重大選擇（如轉職、考試、感情）的使用者。
- 對傳統算命、抽籤、星座、占卜感興趣的年輕族群。
- 習慣數位支付，願意在線上進行捐獻（香油錢）的信眾。

### 核心價值主張
提供一個快速、方便且具備保存功能的線上算命平台，並整合了傳統信仰中「捐香油錢」的機制，讓使用者在獲得指引的同時也能進行線上回饋與祈福。

---

## 2. 功能需求

以下為系統的五項主要功能與對應的使用者故事：

1. **算命、占卜、抽籤**
   - **作為** 一般使用者，**我希望** 能夠在線上進行多種不同的占卜與求籤（如擲筊、抽靈籤），**以便** 獲得對當前困惑的解答與指引。
2. **儲存算命結果**
   - **作為** 註冊會員，**我希望** 系統能夠自動或手動儲存我過去的抽籤與算命結果，**以便** 我日後回顧與追蹤事情的發展。
3. **捐香油錢**
   - **作為** 體驗過服務的使用者，**我希望** 能夠透過線上支付的方式捐款（香油錢），**以便** 對神明表達感謝，或作為平台維護的支持。
4. **會員註冊與登入**
   - **作為** 想要持續追蹤運勢的使用者，**我希望** 能有一個專屬的會員帳號，**以便** 跨裝置查看我的歷史算命紀錄和捐獻紀錄。
5. **籤詩詳解與分享**
   - **作為** 剛抽完籤的使用者，**我希望** 能看到詳細的白話文籤詩解說，並且有按鈕可以一鍵分享到社群媒體，**以便** 我能快速理解神明指示並與朋友分享近期運勢。

## 3. 非功能需求

### 技術限制
- **前端技術**：HTML, CSS (Vanilla CSS), JavaScript, 採用 Jinja2 模板引擎進行頁面渲染。
- **後端技術**：使用 Python 的 Flask 框架作為主要 Web 伺服器。
- **資料庫**：使用 SQLite 儲存使用者資料、算命紀錄與籤詩資料庫，確保輕量且易於部署。

### 效能與安全考量
- **響應式設計 (RWD)**：系統介面需相容手機與桌機瀏覽，提供流暢的跨裝置體驗（尤其手機端抽籤的互動體驗）。
- **資料安全**：會員密碼需經過雜湊 (Hash) 處理後再存入資料庫，避免明文外洩。
- **防止連續點擊**：抽籤與擲筊功能需加入前端防抖 (Debounce) 或冷卻時間，避免伺服器短時間內收到大量請求。

## 4. MVP 範圍 (Minimum Viable Product)

| 層級 | 功能描述 |
| :--- | :--- |
| **Must Have (必須有)** | - 核心抽籤功能（點擊按鈕隨機產生籤詩）<br>- 顯示對應的籤詩內容與白話文詳解<br>- 使用者登入/註冊功能<br>- 儲存算命結果到個人頁面 |
| **Should Have (應該有)** | - 模擬或真實串接捐香油錢功能（先求有紀錄與畫面）<br>- 歷史算命紀錄查詢清單<br>- 基本的 RWD 響應式網頁設計 |
| **Nice to Have (加分項)** | - 動態抽籤特效（如搖籤筒動畫、擲筊動畫）<br>- 社群媒體分享功能（產生運勢圖卡）<br>- 每日運勢提醒或推播 |
